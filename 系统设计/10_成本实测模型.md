# 成本实测模型（背诵检测为主 + 全系统）

> 生成日期：2026-06-06（第 5 轮，解 `01` P0/P1-1 的"成本未落实数"）
> 单价来源：claude-api 技能拉取的**权威实价**（非臆造）。USD→CNY 按 ~7.2 估（汇率浮动，结果给区间）。
> 方法：把设计里的真实调用量（`03`/`06` 检测形态 + `02` 三杠杆）套进实价。假设全部列出，便于云挑战。

## 1. 实价表（每百万 token，USD）

| 模型 | 输入 | 缓存写(5min,1.25×) | 缓存命中(0.1×) | 输出 |
|---|---|---|---|---|
| **Opus 4.8** (`claude-opus-4-8`) | $5.00 | $6.25 | $0.50 | $25.00 |
| **Sonnet 4.6** (`claude-sonnet-4-6`) | $3.00 | $3.75 | $0.30 | $15.00 |
| **Haiku 4.5** (`claude-haiku-4-5`) | $1.00 | $1.25 | $0.10 | $5.00 |

- **输出永远全价**（缓存只省输入）→ 输出 token 是主成本变量。
- Opus **adaptive thinking 的思考 token 计入输出**、按 $25/M 计 → 评分/答疑越"想得多"越贵。
- **Batches API = 全部 5 折**（非实时场景，如题库预生成可用）。
- 最小可缓存前缀：Opus/Haiku 4096 token，Sonnet 2048。

## 2. 背诵检测 · 单次评分成本

**模型分层（`02` 杠杆B）是关键**：L1 走规则匹配=0 API；只有 L2/L3/区分走 Opus 评分。

**单次 L2/L3 Opus 评分**（假设）：
| 项 | token | 单价 | 成本 |
|---|---|---|---|
| 缓存命中（评分 rubric+规则） | 12K | $0.50/M | $0.006 |
| 新输入（grep 教材片段 1.5K + 题 0.3K + 作答 0.4K + grep 工具往返 ~2K） | ~4K | $5/M | $0.020 |
| 输出（评分+证据卡+thinking ~1.5K） | 1.5K | $25/M | $0.0375 |
| **合计** | | | **≈ $0.064 / 次** |

**L1 评分**：规则匹配 = **$0**（Haiku 兜底约 10%，每次 ~$0.002，可忽略）。

## 4. 题库预生成（L2/L3 变体，防背题）

- ~700 个需 L2/L3 的考点（刑民为主；法综多 L1 用规则生成）× 3 变体 ≈ 2100 题。
- 每题：缓存读教材 ~$0.0015 + 输出 0.8K × $25/M = $0.02 ≈ $0.022；**走 Batches 5 折 ≈ $0.011**。
- **一次性 ≈ $23**（批量）；按懒切分摊到 ~4 个月 ramp ≈ **$5–10/月，之后归 0**。

> 题库生成**不是吞金兽**（批量+复用），`02` 杠杆C 有效。

## 5. 答疑 · 偶发，非每日（2026-06-06 云更正）

> ⚠️ **更正**：原按"1 小时/天"估，错。云澄清：**答疑只在"想不明白的题"才找，频率不高**。

引导式=多轮；会话内 prompt caching 命中（5min TTL）。

**单轮（会话中段）**：缓存读历史 ~15K×$0.50/M=$0.0075 + 新输入 ~4K×$5/M=$0.020 + 输出(引导+证据卡+thinking ~1.2K)×$25/M=$0.030 ≈ **$0.058/轮**。

- 偶发：假设 **8–12 次/月、每次 ~15 轮** → 120–180 轮/月。
- **答疑月成本 ≈ $7–10 ≈ ¥50–100/月**（原估 ¥250–500，砍 ~70%）。

> **主成本驱动从"答疑"变回"背诵检测"**——背诵每日、答疑偶发。

## 6b. 按风险分配模型（云定：质量优先，只在真·大差不差处平替）

> 决策原则（2026-06-06）：**需要高质量处尽量用最强模型；只有某场景 Opus 与更便宜模型"大差不差"才平替。**
> ⚠️ **收回**原"背诵 L2/L3 评分用 Sonnet 省 40%"的建议——评分放水=假掌握（断裂点4，云#1 风险），评分必须 Opus。

| 任务 | 质量要求 | 模型 | 理由 |
|---|---|---|---|
| **L2/L3 评分** | 🔴 最高 | **Opus，不降级** | 放水→假掌握，毁掉"客观检测"根 |
| **答疑** | 🔴 最高 | **Opus** | 核心质量；且低频，无成本压力 |
| 拍照 OCR 题干 | 🟡 中 | Claude 视觉 | 题干错→全错 |
| 题库生成 | 🟡 中 | Opus（或 Sonnet+人审） | 烂题→烂检测，可审 |
| L1 关键词匹配 | 🟢 低 | 规则/Haiku | 机械匹配，真·大差不差 |
| 草稿（候选弱项/心得/摘要） | 🟢 低 | Haiku | 云确认把关，错了能抓 |

→ **花大钱处（评分/答疑）= 要质量处，二者一致；能平替处（L1/草稿）本就便宜。质量优先与成本可控不冲突。**

## 6c. DeepSeek V4 作备选的评估（2026-06-06）

> ⚠️ 诚实声明：**DeepSeek V4 确切规格/定价我无法核实（可能晚于知识时点），不臆造数字。** 以下为架构层判断。

**技术可行**：模型选型本就是配置项（可扩展性已定）；DeepSeek OpenAI 兼容，后端可接。

**三个真实摩擦**：
1. v2.3 纪律（强制 grep / 不放水评分 / 无锚标★）**在 Claude 上验证**，换模型未必同样守——尤其评分可能悄悄放水。
2. **Prompt caching 是 Anthropic 专属**，省钱的 90% 不跟着迁，需重搭。
3. 多一个 provider = 多一套故障模式与脾气。

**能去 / 不能去**：
| 能换 ✅（低风险、人审把关） | 绝不能换 ❌ |
|---|---|
| 草稿/摘要（候选弱项/心得/历史摘要） | L2/L3 评分（放水=假掌握） |
| | 答疑（核心质量，且低频无压力） |

**但**：能换处本就用 Haiku、已很便宜（月几元），换 DeepSeek 省小钱却扛"第二 provider"复杂度，**不划算**。

**建议**：现在**不接**，但**架构留口**；当作**可基准测试的备选**——哪天想让评分更省，**先拿真题样本测 DeepSeek V4 评分严不严（该不过就不过、不放水），通过才换**。这是"大差不差才换"原则的落地：**先验证，再信任**。

### 🔴 与旧估算的对比（警报）
- `00` 旧估"月 40–70 元" → **实测是它的 6–10 倍**。
- 旧估幻觉来源：漏算①输出 token 全价 ②背诵检测调用量 ③答疑多轮 ④Opus thinking token。
- **结论：开工前必须用本模型替换旧估，不能揣着 ¥40–70 的幻觉做决策。**

## 7. 已在设计里的省钱杠杆（验证有效）

| 杠杆 | 不用它的代价 |
|---|---|
| **模型分层**（L1 规则/Haiku，L2/L3/答疑 Opus） | 背诵贵 ~5× |

### 新发现的可调旋钮
1. **effort 降档**：评分/答疑用 `effort: medium`（而非默认 high）压 thinking 输出 token——直接砍输出成本。需测对准确率的影响。
2. **背诵 L2/L3 评分可试 Sonnet 4.6**（$3/$15，比 Opus 省 ~40%）：评分是 rubric+grep 的机械活，比开放答疑严谨度要求低，**Sonnet 可能够用**。答疑仍守 Opus。待测。
3. **日熔断**（`01` P2-3）：设 **$3–5/日 ≈ ¥22–36**、月封顶 ~¥1000，防失控。
