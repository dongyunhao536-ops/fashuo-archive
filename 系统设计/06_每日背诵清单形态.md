# 每日背诵清单形态（刑民一期）

> 生成日期：2026-06-05（第 3 轮，背诵模块 / 清单形态专题）
> 范围：**刑民优先，法综搁置**（决策 2026-06-05）。
> 上位依据：`03`（调度模型/检测）、`04`（数据结构）、`05`（建库）。
> 云的原始诉求："不出卡片，只出**背诵清单**，详细科学" + 对应检测功能。

---

## 1. 清单 = 三段式（对应 03 §3.2 的调度三来源）

```
📅 每日背诵清单 = 【A 到期复习】+【B 新考点】+【C 易混对决】
```
| 段 | 作用 | 来源 | 排序/限量 |
|---|---|---|---|
| **A 到期复习** | 记忆维护（间隔重复主体） | kp_state 中"到期"考点 | 按优先级分 P 降序，占清单大头 |
| **B 新考点** | 进度推进 | 配速器额度（03 §3.4） | 当日额度（到期多则少引入，防过载） |
| **C 易混对决** | 交错练习治混淆 | 易混概念库配对 | **≤2 对/日**（尊重交错阈值，03 §3.5） |

**刑民一期的特殊性**：刑民封顶 L3、背的是"规则层"（罪名构成/民法制度/心得规则），所以清单条目**以"规则+应用"为主，不是死记关键词**。L1 在刑民多是"列出构成要件/规则要点"，重头在 L2/L3。

---

## 2. 清单条目（卡片以外的"详细科学"长这样）

每个考点一个**条目块**，七行信息一目了然——为什么今天背它、背到什么程度、背什么、怎么检测：

```
▢ {考点ID} {考点名}                                    [科目·父考点]
   今日缘由: {到期档/弱项错频/真题频率/新考点}
   档位进度: 封顶{L1/L2/L3} | 当前 L1{✓/到期/未测} L2… L3… | 难度D {n}
   锚点    : {教材锚点/法条} | 关联 {真题题号/心得#/易混文件}
   📖 复习要点: {一句话规则 + 你曾经栽在哪}
   ✍️ 今日检测: {本次要测的档位 + 题型}
```

> **显示规则（2026-06-05 定）**：
> - **默认折叠**——清单里每个考点先只显示一行（`▢ 考点名 [科目·父考点] + 缘由标签`），点开才展开下面六行要点/检测。手机屏省空间、先看全貌再逐条钻。
> - **"今日缘由"只给标签，不露具体 P 分值**（如"到期·弱项·⭐高频"），避免数字干扰，云只需知道"为什么是它"。

- **"今日缘由"**＝把调度公式 P 的构成翻成人话（标签化），让云知道系统为什么挑它。
- **"你曾经栽在哪"**＝直接引用弱项里的错误表现，强化记忆（测试效应+情绪锚定）。
- **"今日检测"**＝清单与检测的接缝：看完要点→立刻提取练习。

## 4. 清单 → 检测 的衔接流程

```
看清单条目"📖复习要点"(编码) → 点"✍️今日检测"(提取练习)
   → AI 出题(L1规则匹配/L2-L3 Opus,见02杠杆B) → 云作答
   → AI 评分(rubric+grep教材,v2.3) → 写 detection_log
   → 更新 kp_state(档位/难度D) → 回写弱项/已强化 → 刷新雷达图
```
- 清单是**编码+提示**，检测才是**记忆主引擎**（03 §3.1 测试效应）。
- 一条考点可"只看不测"（赶时间）或"看完即测"；**只有测过才更新记忆状态**——光看不算掌握。

## 5. 每日容量与时间预算（在职防过载）

- **默认日预算 = 30 分钟**（2026-06-05 定，在职可持续），约 8-10 个考点。
- 容量动态：到期复习量大的日子，**B 新考点额度自动调低**（保维护、缓推进）。
- 云可手动"今天只做 A 段"或"加做 N 个新考点"（半自动，决策 2026-06-05）。
- 超量未完成 → 滚动到次日 A 段（按 P 重新排序，不是简单顺延）。

### 智能截断算法（云定"我来定"，2026-06-05）
当可用时间 T < 当日清单总时长时，按"价值密度 + 保护优先"裁剪到 T：

1. **估时**：每考点按档位估时——L1≈1.5min、L2≈3min、L3≈4min、易混区分≈5min。
2. **保护层（绝不裁）**：到期的**高优先级**考点（弱项关联 / ⭐高频）——这些踩在遗忘曲线临界点，跳过=真实丢分。
3. **裁剪顺序**（从先裁到后裁）：
   - ① **B 新考点**最先延后（进度可缓一天，记忆维护不能缓）
   - ② **A 段里低 P 的到期项**（按 P 升序滚到次日）
   - ③ 仍超时 → 把保留的到期项**降一档检测**（如 L3 当日先只测 L2，省 1min）
   - **C 易混对决尽量保**（直击云头号弱点；除非 T 极短才砍到 1 对）
4. **下限**：T 的实际下限按 30 分钟设计；若云某天主动压到更短，按上面顺序裁，并提示"今日只保住了 X 个高价值考点，N 项已滚明天"。

> 一句话：**时间不够先砍"推进"，死保"维护 + 治混淆"**——因为忘掉已学的比少学新的代价大。
