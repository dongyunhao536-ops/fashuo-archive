# 复习规划模块（「教练」tab）设计

> 生成日期：2026-06-06（第 5 轮后新增功能）
> 缘起：云提出"主动思考能力弱、复习规划不够高效科学"，要系统针对每天实际学习（哪科/哪章/听课还是做题）给指导和规划。
> 规划层面优先级（强制）：① 经验帖蓝本 → ② 真题分析 → ③ 教材。本模块即把 `经验帖蓝本/00_适用于云的方法论提炼.md` 从静态文档变成会对话的活系统。

---

## 0. 一句话定位

**背诵系统管"考点级"微调度（SRS+L1-L3），教练 tab 管"科目/章节/轮次级"宏观规划**——这一层此前完全缺失。云三次失败的病根（经验帖 Rule 6："高分者隐性共性=每日复盘，云三次都缺这一步"）正落在这一层。

```
┌─────────────────────────────────────────────┐
│  📋 教练（宏观层 · 新增）                        │
│   听课/做题/背诵轮次推进 · 阶段切换 · 双轨节奏     │
│   每日复盘 · 周/月检查 · 红线预警                 │
│   ——— 操作单元：科目 / 章节 / 轮次 ———          │
└───────────────┬─────────────────────────────┘
                │ 暴露的卡点↓（候选弱项，接已有待办筐）
┌───────────────┴─────────────────────────────┐
│  📖 背诵（微观层 · 已有）                        │
│   考点级 SRS · L1-L3 检测 · 易混对决             │
│   ——— 操作单元：考点 ———                       │
└─────────────────────────────────────────────┘
```

## 1. 三项已拍板决策（2026-06-06）

| 决策项 | 选择 | 理由 |
|---|---|---|
| **位置/形态** | **独立第 5 个 tab「教练」**（背诵/答疑/教练/仪表盘/弱项） | 对话式日志+规划+复盘自成一页，地位独立不挤其他 |
| **自动化程度** | **系统推荐 + 云拍板** | 与答疑苏格拉底式、背诵"推荐+拍板"一致：系统给经验帖-grounded 的科学框架，云确认/微调，复盘亲自做——**框架由系统兜底（保科学），判断与复盘留给云（练思考）** |
| **覆盖范围** | **只法硕 5 科** | 云定。政英自管 |

**⚠️ 覆盖范围的诚实代价**：选"只 5 科"后，经验帖 **Rule 8 政英红线（政治 9 月启动 / 英语全程 1h）不由本模块提醒**，需云自己盯。本模块的红线（§4）只覆盖 5 科 + 总投入时长。

**为什么不全自动排死计划**（记录被否方案的病根）：①全自动=练不到主动思考，与答疑/背诵立项初衷自相矛盾；②AI 排的死计划易脱离云当天真实状态（加班了/某章卡住了）；③"照做"养成依赖，考场上没人替你规划。故定"推荐+拍板"。

## 2. 核心交互：你丢一句话，系统给四段

### 输入（自然语言，低摩擦）
云每天/每次学完丢一句，例：**"今天刑法第 5 章，听课"**、**"做了 2021 民法真题，错了 4 道"**、**"法理第 3 章背了一遍"**。
- 形态=自然语言（呼应云原话"我今天告诉你我学了哪一科"），AI 解析出 `科目 / 章节 / 形式(听课·做题·背诵·复盘) / 可选(用时·正确率·感受)`。
- 解析结果**回显让云确认**（防解析错），再入库。

### 输出（四段，全部 grounded 经验帖）

| 段 | 内容 | 经验帖依据 | 例（"刑法第5章 听课"） |
|---|---|---|---|
| **① 即时点拨** | 这一步学完，下一动作该是什么 | **Rule 1**（刑民理解为本：听课后必做题验证，不能直接背） | "刑法听完别急着背——先做本章配套题验证真懂了，做题暴露的点丢答疑" |
| **② 进度归位** | 把这章标进"一轮听课进度"，对照轮次表看在不在轨 | **Rule 4**（四轮三阶段表）+ **Rule 2**（9月底基础结业红线） | "刑法一轮推进 5/22 章 · 当前在'第2轮·框架记忆'窗口 · 在轨 ✅" |
| **③ 下一步规划（推荐，可改）** | 明天/本周建议，结合节奏+阶段+弱项 | **Rule 3**（双轨节奏）+ 阶段开关 + 当前弱项 | "明早通勤听第6章音频；今晚 2h 大块留给弱项'担保物权优先顺位'" |
| **④ 复盘提取** | 引导云说"今天最不懂/最有把握"，最不懂→候选弱项 | **Rule 6**（睡前3行：学了啥/最不懂/最有把握） | "今天最卡哪一点？→ 生成候选弱项，收进档案吗？[✓/✗]" |

**"推荐+拍板"落地**：③ 永远是**建议卡**带 [采纳]/[改一改]/[今天不按这个] 三个键；不是命令。云改了，系统记住偏好。

## 3. 周期性（不只是每天）

| 周期 | 动作 | 经验帖依据 |
|---|---|---|
| **每日** | 复盘 3 行（学了啥/最不懂/最有把握） | Rule 6 |
| **每周日** | 周复盘卡：本周实际投入 vs 计划、最不懂1-2点入弱项、下周重点（刑民优先） | §8.2 |
| **每月初** | 月检查清单：上月模拟分、进度 vs 轮次表、弱项是否下降、节奏是否要调 | §8.1 |

---

## 4. 红线预警（§8.3，本模块只覆盖 5 科 + 总时长）

| 触发条件 | 系统行为 |
|---|---|
| 9 月底模拟 < 320 | APP 内预警：目标可能偏离，需重评投入（**非劝退，是主动调整窗口**） |
| 连续 2 周投入 < 20h | 提醒：跌出在职模板节奏 |
| 同一弱项错 ≥3 次 | 提示转专题攻克，暂缓推进 |
| 基础阶段 4 项（Rule 2）8月底缺2 / 9月底缺1 | 预警 / 降级（战略放弃某章） |

> 政英相关红线（Rule 8）**不在本模块**——覆盖范围选了只 5 科。

## 5. 数据模型（接 `11` 的"待办筐+PC登记员"，零新架构冲突）

```
-- B 运行状态（Supabase 唯一真相，不回 markdown；与 detection_log 同类=流水）
study_log(id PK, 日期, 科目, 章节, 形式(听课/做题/背诵/复盘),
          用时分, 正确率, 自评感受, 解析原文, created_at)

-- 轮次/阶段进度（聚合视图，可由 study_log 实时算，或单独存）
syllabus_progress(科目, 章节总数, 一轮已听课章数, 当前轮次, 当前阶段模式)
```

- **学习日志=流水**，与 `detection_log` 同性质 → 存 Supabase B 类、**不回 markdown**（呼应 `11` §1"运行状态不进 markdown"）。
- **复盘提取的"最不懂"→候选弱项** → 走**已有待办筐** `proposals`（type=弱项，来源=复盘）→ 云手机确认 → PC 登记进 `当前弱项.md`。**不新建闭环，复用 `11`。**
- 与 `进度记录/学习进度.md` 关系：该 markdown 是 Rule 6 的人读档；PC 登记时可选把 study_log 汇总写入（内容层），但**流水本身不进**。

## 6. 与已有模块的接口（强调"复用不重造"）

| 接口 | 复用什么 |
|---|---|
| **阶段切换** | 直接复用背诵调度的**全局模式开关**（`00` 决策3：6月铺开→9月高频→考前2周）。教练 tab 是这个开关的"人类可读面板"，不另立一套阶段逻辑 |
| **弱项沉淀** | 复用 `11` 待办筐，不新建路径 |
| **进度/雷达** | 教练的"轮次进度"喂仪表盘 Hero/周热力；雷达仍由背诵 L1-L3 聚合（两者正交：教练=覆盖广度，背诵=掌握深度） |
| **答疑** | 即时点拨里"丢答疑"=跳答疑 tab；答疑暴露的卡点也回流复盘 |

## 7. 开放参数（外置配置，呼应"可扩展性原则"）

- **轮次表日期**（Rule 4：1轮7-8月/2轮8-9月/3轮10-11月/4轮12月）→ 配置化，云可按真实进度滑动。
- **各科章节总数**（算"X/22 章"进度的分母）→ 建库时从教材目录抽，配置化。
- **红线阈值**（320 分 / 20h/周 / 错3次）→ 配置化。
- **双轨节奏模板**（Rule 3 工作日早/通勤/晚 + 周末）→ 配置化，云可填自己真实可用时段。
