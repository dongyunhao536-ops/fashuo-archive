# 极简暗色版 UI 方案 · 审查优化（2026-06-10）

> 对象：`极简暗色版.html`（2026-06-09 20:20 改版，Apple HIG 暗色 / systemBlue 单强调色 / 8 屏）。
> 审查方法：对照 app 现有 13 页面 + 7 交互组件逐屏核对功能面。结论：**风格方向保留，结构按下列 13 条修正后实施**。

## 一、缺屏（方案 8 屏 → 实际需 12 屏）

1. **缺 /duel 易混对决**（三态：背辨析档案 → 对决作答 → 评分揭晓）。清单里只画了一行入口，但这是独立完整流程，且 6/10 刚扩容到 56 对、改为"先背后测"。
2. **缺 /cards 全卡浏览 + 单卡页**（6/10 新增）：863 张 Anki 卡按卡组章节树兜底可达，含法条卡。
3. **缺 /weak 弱项页**：tabbar 第 5 个 tab 就是"弱项"，方案却没画屏。需：科目筛选 + 错次 + 三档 L1/L2/L3 状态 + 锚点 + 到期。

## 二、过时屏（按 6/10 改版后的真实结构重画）

4. **背诵清单**：方案画的是「复验/到期/新考点」三段，已过时。现实结构 = 「🌱新背诵 / ⏰待复习」两页签（segmented control）+ 科目筛选（全部/五科 pill）+ 背诵量 10/30/50 + **科目→章→节分级折叠**（带章节号）+ 易混段（每科每日 5 对）+ 全卡浏览入口。
5. **进度环砍掉**：环的数据源（done/total）不在 getTodayPlan 里，页签计数已承载同样信息；完成度留在仪表盘 hero。不为装饰加数据查询。

## 三、补漏（方案画了壳但缺现有功能件）

6. **仪表盘**补：五科雷达（改单色：蓝线+灰网格）、本周活动热力（灰阶→蓝渐进）、今日已学分钟/检测次数。
7. **答题屏**：编码阶段实际是 **Anki 原卡 HTML 白底纸卡**（保真红线：颜色排版一字不差，恒白底不反色）——黑底上作"纸卡"呈现：白卡 12px 圆角，无额外装饰。方案的 编码/作答/评分 segmented 采纳为**阶段指示器**（不可点）。
8. **评分屏**补：缺失要点（✗ 红）、G1 弱项投递提示、「再测一次」按钮、证据卡 grep 行号。
9. **答疑屏**补：科目选择 pill 行（方案 nav 右上单徽章不够）、候选沉淀提示（🗂 N 个候选进待办筐）、信心度 <70 提"建议核对标答"、费用/grep 命中元信息条。
10. **教练屏**补：空态示例 prompt 按钮、解析回显条（"我理解为…不对就换种说法"）、红线预警条（红 tint）。三键采纳已画 ✓（已接 plan_decision 回写）。

## 四、全局修正

11. **tab 顺序采纳方案**：今日 / 背诵 / 答疑 / 教练 / 弱项（首页挪第一位；现实现"仪表盘"在第 4 位，改）。图标全换 SF 单线风 SVG，去 emoji。
12. **颜色纪律修正**：方案原则"状态色仅评分结果"过严——频率高/中/低、复验/到期、L 档位是调度关键信号，全灰阶损失可扫读性。修正为：**正文/标题/图标严格灰阶 + systemBlue 唯一强调；信息徽章允许 18% tint pill（蓝/绿/橙/红）；大面积状态色仍仅评分结果**。
13. **字体与主题**：Geist（仅拉丁字形，中文场景无意义）→ `-apple-system, "PingFang SC"` 系统栈；**恒暗色单套**（删 prefers-color-scheme 双套与全部 dark: 变体），themeColor #000；Anki 纸卡是唯一白底区域。
