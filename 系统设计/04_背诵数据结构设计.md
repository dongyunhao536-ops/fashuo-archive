# 背诵引擎 · 数据结构设计

> 生成日期：2026-06-05（第 3 轮，背诵模块 / 数据结构专题）
> 目标：定义"考点"长什么样、SRS 状态存哪、检测怎么记，以及**用一个"锚点规范化层"把考点 ⇄ 弱项 ⇄ 心得 ⇄ 易混 ⇄ 真题钉死**。
> 上位依据：`02`（真相源分层：内容→markdown，易变→Supabase）、`03`（调度模型与闭环飞轮）。

---

## 1. 三层数据划分（各归各位，避免冲突）

| 层 | 内容 | 变动频率 | 落地 | 写者 |
|---|---|---|---|---|
| **① 考点定义层** | 考点是什么、关联哪些档案 | 低（建库+偶尔校正） | **markdown @ GitHub** | PC 端 |
| **② 学习状态层** | 间隔档/难度 D/到期日/三档掌握度 | 高（每天每次检测都变） | **Supabase** | 手机端 |
| **③ 检测记录层** | 每道检测题的流水 | 高（流水） | **Supabase** | 手机端 |

三层用 **考点ID** 串起来。定义层是内容真相源（可 grep、可人读、进 git）；状态/记录层是易变数据，手机直接写，根除双写冲突（呼应 `02` 决策 D-7）。

---

## 2. 考点ID 命名规范（懒切分下也稳定）

```
考点ID = {科目码}-{4位序号}    例：XF-0142
科目码：XF刑法 MF民法 FL法理 XZ宪法 LS法制史
```
- 序号**分配后永不复用、永不重排**——即使"按需懒切分"（P2-1）后续才精校某考点，ID 也不变。
- ID 只是稳定主键，人读靠"考点名"字段。
- 与 CLAUDE.md 既有 `知识点索引.md` 约定的协调：**`知识点索引.md` = 考点定义层 + Supabase 状态层 每日同步脚本生成的"只读视图"**（`概念名 | 页码 | 掌握程度 | 最后复习日`正好是两层 join 的结果）。

---

## 3. 考点定义层字段（markdown，每科目一个文件 `考点库/刑法.md` …）

采用**块格式**（非宽表）——13 个字段用表会爆，块格式可 grep 可扩展：

```
### XF-0142 高空抛物罪
- 父考点: 妨害社会管理秩序罪/扰乱公共秩序   # 用于雷达图聚合
- 教材锚点: 法律更新档案/01_刑法相关更新.md（刑修十一§291之二；主教材未覆盖）
- 类型: 应用            # 纯记忆/理解/应用 → 决定封顶档 L1/L2/L3
- 考法: 客观题 + 案例题   # 决定检测题型：主观题→默写，客观题→辨析
- 真题频率: 中           # 高3/中2/低1 + ⭐，来自高频考点文件
- 关联真题: —（模拟卷一案例57）
- 关联心得: 刑法#20
- 关联弱项: 高空抛物罪盲区（卷一案例57）
- 关联易混: 刑法-高空抛物vs过失致死vs过失危害公共安全.md
- 关联法律更新: 法律更新档案/01_刑法相关更新.md
- 难度基线: 6           # 默认5，弱项/新法/反直觉点上调
```

字段说明：
- **父考点**：节/标题级，雷达图按此聚合（`03` § 3.2）。
- **类型 → 封顶档**：纯记忆点封顶 L1（如"铸刑书=子产"），理解点 L2，应用点 L3。
- **考法 → 检测题型**：主观题考点 L1 也用生成式默写（匹配手写考场，`03` 断裂 5）。
- **难度基线**：建库初值，之后被检测结果动态覆盖（存 Supabase，见 § 4）。

## 4. 学习状态层字段（Supabase 表 `kp_state`）

| 字段 | 类型 | 说明 |
|---|---|---|
| kp_id | text PK/FK | 考点ID，关联定义层 |
| 当前档 | enum(1,3,7,15,30) | 当前间隔档（天） |
| 难度D | float(1-10) | 动态，均值回归（`03` § 3.2，防 ease hell） |
| 上次复习日 / 下次到期日 | date | 算到期度 |
| L1状态 / L2状态 / L3状态 | enum(未测/通过/未过) | 三档独立 |
| L1分 / L2分 / L3分 | float | 各档最近得分 |
| R_target | float | 该考点目标留存（按价值算，可缓存，`03` § 3.3） |
| 当前优先级分 | float | 每日 6 点重算缓存（`03` § 3.2 的 P） |
| 累计复习次数 / 累计错误次数 | int | 统计 + 难度调节 |
| 已掌握 | bool | 三档全过且未过期 |

## 5. 检测记录层字段（Supabase 表 `detection_log`）

| 字段 | 说明 |

## 6. 锚点规范化层（把所有档案钉到一起的关键）

现有弱项锚点格式混乱，必须规范化成统一三元组才能自动关联：

```
锚点 = { 类型, 源, 定位 }
类型 ∈ {教材行号, 法条, 司法解释, 档案文件, 无明文}
源   ∈ {考试分析, 民法学, 刑法典, 民法典, 法律更新档案, …}
定位  = 行号range | 条号 | 文件路径 | null
```

真实弱项锚点 → 规范化示例：
| 原始（来自 `当前弱项.md`） | 规范化 |
|---|---|
| 考试分析 第293行 | {教材行号, 考试分析, 293} |
| 考试分析 第229-236行 | {教材行号, 考试分析, 229-236} |
| 民法学 第970-972行 | {教材行号, 民法学, 970-972} |
| 民法典416条 | {法条, 民法典, 416} |
| 法律更新档案/01_刑法相关更新.md | {档案文件, 法律更新档案, 01_刑法相关更新.md} |
| 教材未明文，按合同相对性立场 | {无明文, –, null} |

**自动关联规则**：
- 考点.教材锚点 与 弱项/心得.锚点，**类型+源相同**时：行号 range 重叠 ／ 条号相等 → 自动建立关联。
- "无明文"类型无法自动关联 → 靠 § 3 的 `关联弱项/关联心得` 字段人工挂（建库时一次性挂好）。

→ 这层是 `03` "闭环飞轮"和"易混交错"的数据地基：没有规范化锚点，飞轮转不起来。

## 7. 一个考点贯穿全档案（闭环数据路径示例）

以 **XF-0142 高空抛物罪** 走一遍飞轮（`03` § 2、`02` Part 2）：

```
[答疑/检测] 云在 L3 案例检测把"高空抛物"误答成"以危险方法危害公共安全罪"
   │  detection_log 记一条：kp=XF-0142, 档=L3, 通过=否, grep命中=法律更新档案/01
   ▼
[弱项] 通过"关联弱项"字段定位到"高空抛物罪盲区" → 错误频率+1
   │  （锚点 {档案文件,法律更新档案,01} 命中）
   ▼
[易混] "关联易混"→ 刑法-高空抛物vs过失致死vs过失危害公共安全.md
   │  登记一次混淆记录；下次调度**把这对配同一天**做区分题（交错，`03`§3.5）
   ▼
[调度] kp_state: 难度D↑、L3 退半档→L2重测、优先级分↑（价值V含弱项加权）
   ▼
[检测] 重测 L2→L3 通过 → 回写"已强化项"、已掌握=true
   ▼
[雷达图] 按父考点"妨害社会管理秩序罪"聚合，刑法掌握度上调
```

## 8. 冷启动建库流程（考点库怎么长出来）

1. **抽骨架**：结构规则扫 `教材/考试分析_文本.txt` 的标题行（带"（P11）（2019…）"的行）→ 父考点 + 子考点 + 教材行号锚点 + 真题年份。
2. **填高频元数据**：合并 `真题分析/0X_高频考点.md` 的 bullet → 真题频率 + 关联真题题号。
3. **挂关联**：用 § 6 锚点规则，把 弱项/心得/易混/法律更新 自动挂到考点；"无明文"的人工挂。
4. **补冷点**：高频文件没覆盖的标题 → 仍建考点，真题频率=低（`03` 断裂 5，防漏冷点）。
5. **按需懒切分**：骨架先全建，**精校（核对锚点、定类型/考法/难度）推迟到调度命中该考点时**（P2-1，摊薄 2-3h 集中校正）。
6. **云验收**：每精校一批，云抽查锚点是否对、类型/考法判得对。
