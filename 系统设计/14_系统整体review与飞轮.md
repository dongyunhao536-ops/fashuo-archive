## 1. 框架重定义：不是"5 个平级模块"，而是 3 动 + 1 账本 + 1 仪表

把五模块当平级，就会做成 5 个互不相干的页面（=云担心的"相互独立"）。正确结构：

```
   📖背诵(T0)      💬答疑(T0)      📋教练(T1)     ← 三个"动作"
       \              |              /
        ▼             ▼             ▼  （存信号 / 取信号）
   ┌────────────────────────────────────┐
   │  共享账本 = 系统的复利资产              │
   │  弱项档案 + kp_state掌握档 + 心得文件    │
   └────────────────────────────────────┘
                     │
                     ▼
        📊仪表盘 + ⚠️弱项页 ← 账本的两块"表盘"（只读），非独立模块
```

- **三个动作**的价值不在各自页面，而在**都往同一账本沉淀信号、又都从账本取信号**。
- **仪表盘/弱项页是表盘**（账本的视图），不是第 4、5 个模块。
- **"越用越强" = 账本信号越用越密 + 三个动作读它越来越准**。
- 整机 review 的真问题只有一个：**三个动作，是否每一个都既喂账本、又取账本？哪里断了？**

---

## 2. 飞轮该怎么转：每个动作既是信号源也是信号汇

| 动作 | 喂账本（信号源） | 取账本（信号汇） |
|---|---|---|
| 📖背诵 | detection_log→kp_state(掌握档)；**反复失败→候选弱项(G1)** | 弱项错频→调度权重；kp_state→跳过已掌握 |
| 💬答疑 | 候选弱项+候选心得→待办筐；会话摘要(卡点) | 心得/真题/教材(grep)；Top5弱项+未解决卡点(注入) |
| 📋教练 | study_log；复盘候选弱项；轮次进度 | 经验帖；当前弱项(攻坚建议)；阶段开关 |

**真正在复利的资产**（"越用越强"具体指什么）：
1. 弱项错频累积 → 调度越来越精准打痛点
2. kp_state 掌握档 → 越背越知道哪些稳了、不再浪费时间
3. 心得文件增长 → 答疑/检测判断越来越准（锚点更多）
4. 会话摘要 → 答疑连续性，不重复教
5. 教练轮次校准 → 越用越贴合真实进度

## 3. 六个集成缺口（现在是"相邻"，不是"咬合"）

| # | 断点 | 现状 | 不补的后果 | 优先级 |
|---|---|---|---|---|
| **G1** | 背诵反复失败 → 弱项 | kp_state 有累计错误但不流向弱项档案 | 背诵测出的痛点，答疑/教练看不见 | 🔴 开工前 |
| **G2** | 答疑澄清 → 背诵复验 | 答疑"懂了"后无机制安排背诵复验 | 讲懂≠记住，测试效应漏掉，白讲 | 🔴 开工前 |
| **G3** | 教练日志 vs 背诵/答疑自动活动 | 教练手动 self-report，背诵/答疑已自动记 | 双重录入 + 教练盲视 APP 内真实活动 | 🟡 二期(schema 先留口) |
| **G4** | 雷达只来自背诵 | 雷达=背诵 L1-L3 聚合，答疑/教练覆盖没进 | 答疑会、没背过的考点显示 0%，失真 | 🟡 二期 |
| **G5** | 心得写入端偏窄 | 仅"答疑→候选心得"，背诵易混/教练复盘没接口 | 核心资产喂养渠道窄 | 🟢 优化 |
| **G6** | 会话摘要 vs 弱项 边界 | 两者都记"你不会啥"，`12` 设计了隔离 | 不钉死会重复计数/互相污染 | 🟡 落 schema |

**G1+G2 是 T0 两核之间的断点 → 必须开工前接上**，否则背诵答疑=两座共用数据库的孤岛。

## 4. 三视角分析

### 4.1 🏗️ 软件架构师
- **骨架对**：`11` 三类数据各有主人 + 待办筐=事件溯源雏形，append-only 流水天然支撑"可观测的越用越强"。
- **升级点**：从"共享数据库式集成"升到"显式握手"。把待办筐 `proposals` 泛化为轻量**事件表 `events`**（单用户一张 Supabase 表+轮询足够，不上消息队列）：背诵失败→`event(弱项候选)`；答疑澄清→`event(复验请求,kp_id)`→背诵消费。这解 G1/G2/G3。
- **可解释面板**（"为什么今天排这些"）必须有，否则单用户参数漂移黑箱化，无法判断是"越用越强"还是"越用越偏"。
- **冷启动+稀疏**：不依赖统计/ML，规则+配置兜底（已定），守住。

### 4.2 ⚖️ 法硕教授
- **飞轮学习科学正确**：测试效应+间隔重复+苏格拉底+复盘，组合有据。
- **G2 是硬伤不是优化**：测试效应要求"理解后必须一次主动提取才固化"。答疑讲懂不复验=做一半。必补。
- **收敛方向必须是真题靶心**：w1(真题频率)数值要给够，否则飞轮收敛到舒适区（=`09` 指出的"强项偏离靶心"复发）。
- **刑民/法综两条转速不同的飞轮**：法综=背诵检测主导，刑民=答疑理解主导(背诵只背规则层)。不能一套权重套两科。
- **最大反向风险=越用越自欺**：检测太松/AI 放水→假掌握。守门=L2/L3 真题锚定+v2.3 不放水（🔴 红线）。破了则飞轮=自欺机器。
- **主观题刻意练习偏弱**：成段写作(Rule5 概念+分点/三段论)缺专属落点。北大拉分关键，建议飞轮里给明确位置。

### 4.3 🎓 备考学生（云）
- **怕"科学但累"**：G3 双重录入是真痛点 → 教练日志自动吃背诵/答疑活动，只补 APP 外的(听课/看书/纸质做题)。
- **确认疲劳**：高置信自动收、低置信才问 + 批量确认。
- **冷启动要诚实**："还在了解你"提示，别假装精准，否则看它不准就弃用。
- **"越用越强"要可见**（否则没动力喂它）：①越懂我哪弱(Top5在动/错频降) ②越不浪费时间背会了的(已掌握少打扰) ③答疑接得上上次(上次卡 X 接着讲)。
- **别唠叨**：红线预警克制，按真红线触发，不天天弹"你落后了"。

## 6. 开工清单（写代码前必须钉死的集成契约）

> "越用越强"是架构属性，不是后期能加的功能。**账本和事件契约必须 day-1 就在**，模块可增量上线。

### 必须 day-1 落地（否则模块会长成孤岛）
1. **共享账本 schema 定稿**：弱项档案(markdown)+kp_state+心得，去重键=科目+知识点（`11` 已起草）。
2. **事件表 `events`**（泛化待办筐）：至少含 `弱项候选 / 心得候选 / 复验请求` 三型 → 解 G1/G2/G3 的接口先留好。
3. **G1 接线**：背诵检测连续失败 N 次/退档 → 自动发 `弱项候选` event。
4. **G2 接线**：答疑收口时若卡点对应 kp_id → 发 `复验请求` event → 背诵下次清单插入该考点检测。
5. **可解释字段**：detection_log 存 grep 命中行号/信心度/模型（已设计），调度产出附"为什么排它"。
6. **真题锚定 L2/L3 题源**（🔴 红线）：评分引擎不放水，自评≤0.5 次。

### 可二期（schema 先留口，不 day-1 实现）
- G3 教练日志自动聚合背诵/答疑活动（先手动，schema 留 `source=auto/manual`）
- G4 雷达融合答疑+教练覆盖（先纯背诵聚合，留"未学/学过未测/已测"三态字段）
- G5 心得多源投递、G6 摘要/弱项边界落 schema

### 推荐 build order（飞轮优先，非模块优先）
> 原则：先把 **T0 飞轮的最小闭环**跑通，再加 T1，仪表盘/弱项页随之长。

1. **底座**：共享账本 schema + 事件表 + grep 工具链（search_textbook/xinde/zhenti）+ 缓存。
2. **答疑 v2.3 直答版**（`12` 已定先做）：跑通管线，顺带产出评分引擎（背诵 L2/L3 复用）。
3. **背诵刑民**（`00` 已定一期刑民）：L1 秒判+L2/L3 检测，接 G1/G2 → **此刻 T0 飞轮首次闭环**。
4. **弱项页 + 仪表盘**：账本一旦有信号就有得显示（不是独立开发，是账本的视图）。
5. **教练（T1）**：宏观层叠加，接复盘→弱项、读阶段开关。
6. 引导式答疑、OCR、法综建库、教练自动聚合=后续迭代。
